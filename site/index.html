Best-in-class workflows
for operators and agents

Great AI services are built on great infrastructure. KAZI OS turns your domain blueprint into production-ready workflows, structured reports, and client-specific delivery — so expert teams can scale what they know without rebuilding how they work.

Get started
M manifest.yaml
T tenant.yaml
P pipeline.py
# domain manifest — your expertise as config
domain:
name: content
version: 1.0.0

pipelines:
produce:
stages:
- agent: content_scout
- agent: content_scorer
- agent: content_compiler
trigger:
type: scheduled
cron: "0 6 * * 1,4"
# tenant config — one per client
org_id: silicon-xchange
name: Silicon Xchange
domain: content

destinations:
review:
adapter: notion
config:
workspace: ${NOTION_WORKSPACE}
publish:
adapter: webhook
config:
url: ${PUBLISH_WEBHOOK}
# what KAZI OS generates from your manifest
from kazi import Pipeline, run

pipeline = Pipeline.from_manifest(
"./manifest.yaml"
)

# run for a specific tenant
result = await run(
pipeline,
tenant="silicon-xchange"
)

print(result.status) # "completed"
