Your methodology becomes the template.
Every output inherits it.

Define scoring rubrics and delivery templates once. KAZI OS generates structured, client-ready deliverables on every pipeline run — consistent, auditable, and versioned.

Read the docs

How it works

01

Define your scoring rubric

Declare dimensions, weights, and tier thresholds in scoring.yaml. This is your methodology as structured data.

02

Create delivery templates

Write Jinja2 templates that reference scoring data, tenant context, and pipeline outputs. One template, infinite outputs.

03

Generate on every run

The TemplateRenderer produces HTML, Markdown, JSON, or PDF — automatically, on every pipeline completion.

Scoring rubrics in YAML

Multi-dimensional weighted scoring with configurable tiers. Change the rubric, every future output reflects the new methodology.

  • Unlimited dimensions with custom weights
  • Tier thresholds route outputs (Publish / Revise / Rethink / Discard)
  • Scores are deterministic and auditable
  • Rubrics version with your domain — methodology evolves, history preserved
S scoring.yaml
dimensions:
relevance:
weight: 0.30
description: Alignment with audience needs
depth:
weight: 0.25
description: Technical substance and insight
originality:
weight: 0.25
description: Novel perspective or framing
actionability:
weight: 0.20
description: Clear next steps for the reader

tiers:
- name: Publish min: 0.80
- name: Revise min: 0.60
- name: Rethink min: 0.40
- name: Discard min: 0.00

Templates with full context

Jinja2 templates have access to scoring results, pipeline outputs, tenant metadata, and run history. One template produces infinite variations.

  • Access scoring breakdown per dimension
  • Conditional sections based on tier classification
  • Tenant-specific branding and context injection
  • Multi-format: HTML, Markdown, JSON, PDF
T templates/report_v1.html
<h1>{{ tenant.name }} — {{ report.title }}</h1>

<section class="scoring">
{% for dim in scoring.dimensions %}
<div>
{{ dim.name }}: {{ dim.score }}/10
<!-- Renders progress bar -->
</div>
{% endfor %}
</section>

{% if scoring.tier == "Publish" %}
<div class="approved">
Ready for distribution.
</div>
{% endif %}

Structured output, every run

Every pipeline run produces a structured output file with full scoring breakdown, tier classification, and metadata. Query it, compare it, benchmark it.

  • JSON output with complete scoring data
  • Rendered HTML/PDF for client delivery
  • Decision log: why this score, why this tier
  • Benchmark against historical runs
$ kazi render --run run_2026_05_26_001 --format pdf

▶ Scoring: 4 dimensions, weighted total = 0.84

Built for consistency

Multi-Dimensional Scoring

Unlimited weighted dimensions. Each scored independently, combined into a single weighted total.

Tier Classification

Configurable thresholds route outputs to different actions. Publish, revise, rethink, or discard.

Template Engine

Jinja2 with full pipeline context. Conditionals, loops, filters. One template, infinite outputs.

Multi-Format Output

HTML, Markdown, JSON, PDF. Same template, different renderers. Choose per tenant or per run.

Methodology Versioning

Rubrics evolve with your domain. History preserved. Compare outputs across methodology versions.

Audit Trail

Every score has a reason. Every tier has a threshold. Full decision log for compliance and review.
