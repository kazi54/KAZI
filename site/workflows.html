⚡ Workflows

Define once. Run everywhere.
Scale without rewriting.

Your domain expertise becomes a manifest. KAZI OS reads it and builds the pipeline — agents, stages, triggers, retries, and human-in-the-loop checkpoints. No orchestration code required.

Read the docs

How it works

01

Define your manifest

Declare your agents, pipelines, stages, and triggers in a single YAML file. This is your domain expertise as config.

02

KAZI OS builds the pipeline

The PipelineBuilder reads your manifest, discovers agents from your domain directory, and wires the orchestration automatically.

03

Run, pause, resume

Execute on-demand or on schedule. Pipelines pause at human checkpoints, wait for approval, then resume automatically.

Define your workflow in YAML

No Python orchestration code. No DAG builders. Just declare what agents run, in what order, with what triggers.

  • Agents are referenced by name — auto-discovered from your domain
  • Stages run sequentially with typed input/output contracts
  • Human checkpoints pause the pipeline until approved
  • Triggers fire on schedule, on-demand, or on event
M manifest.yaml
pipelines:
produce:
description: Weekly content production
stages:
- agent: content_scout
- agent: content_scorer
- checkpoint: approve_topics
- agent: content_compiler
- checkpoint: approve_draft
- agent: content_distributor
trigger:
type: scheduled
cron: "0 6 * * 1,4"

Agents are just Python classes

Extend BaseAgent. Implement one method. KAZI OS handles discovery, typing, validation, retries, and logging.

  • Four agent types: research, evaluation, synthesis, delivery
  • Typed input/output contracts with Pydantic
  • Built-in retry with exponential backoff
  • Automatic execution timing and structured logging
A agents/content_scout.py
from kazi import BaseAgent, AgentContext

class ContentScout(BaseAgent):
name = "content_scout"
type = "research"

async def execute(self, context: AgentContext):
# Your domain logic here
sources = await self.scan(
context.config["sources"]
)
return {"items": sources}

Run from CLI, API, or MCP

Four interfaces to the same platform. Choose what fits your workflow — terminal, HTTP, Python, or AI-native.

  • CLI for operators: kazi run produce
  • API for integrations: POST /api/run
  • Python for scripts: await pipeline.run()
  • MCP for AI agents: Claude, GPT, or any model

Built for production

Sequential & Parallel

Stages run in order by default. Use FanOut for parallel execution with concurrency control.

Human-in-the-Loop

Pipelines pause at checkpoints. Approve via CLI, API, Notion status change, or MCP tool call.

Retry & Backoff

Exponential backoff with jitter. Circuit breaker for persistent failures. Configurable per stage.

Scoring System

Multi-dimensional weighted scoring with configurable tiers. Agents score, pipelines route by tier.

Agent Discovery

Drop a Python file in your domain's agents/ directory. KAZI OS finds it, registers it, wires it.

State Persistence

Every run is saved. Resume from any stage. Retry failed runs. Full decision log for auditing.
