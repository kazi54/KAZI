# Templates

Templates define the output format of your reports. They are Jinja2 files that receive scored pipeline data and render it into deliverables.

## File Location

```
your-domain/
└── templates/
    ├── report_v1.html       ← HTML report
    ├── brief_v1.md          ← Markdown brief
    └── export_v1.json       ← JSON export
```

## Template Context

Every template receives these variables:

| Variable | Type | Description |
|----------|------|-------------|
| `pipeline_name` | string | Name of the pipeline that produced this output |
| `org_id` | string | Tenant identifier |
| `run_id` | string | Unique run identifier |
| `date` | string | ISO date of the run |
| `scored_findings` | list | Items that passed through the scoring system |
| `metadata` | dict | Pipeline-level metadata (duration, stage count, etc.) |
| `scoring` | dict | Scoring system config (dimensions, tiers, weights) |

## HTML Template Example

```html templates/report_v1.html
<!DOCTYPE html>
<html>
<head>
    <title>Intelligence Report — {{ date }}</title>
    <style>
        body { font-family: 'Inter', sans-serif; max-width: 800px; margin: 0 auto; padding: 40px; }
        .header { border-bottom: 2px solid #1a1a1a; padding-bottom: 20px; margin-bottom: 40px; }
        .finding { margin-bottom: 32px; padding: 24px; border: 1px solid #e5e5e5; border-radius: 8px; }
        .score-badge { display: inline-block; padding: 4px 12px; border-radius: 12px; font-size: 12px; font-weight: 600; }
        .tier-publish { background: #dcfce7; color: #166534; }
        .tier-revise { background: #fef3c7; color: #92400e; }
        .dimensions { display: grid; grid-template-columns: repeat(2, 1fr); gap: 8px; margin-top: 12px; }
        .dim { font-size: 13px; color: #6b7280; }
    </style>
</head>
<body>
    <div class="header">
        <h1>Intelligence Report</h1>
        <p>Client: {{ org_id }} | Date: {{ date }} | Run: {{ run_id }}</p>
        <p>Pipeline: {{ pipeline_name }} | Items scored: {{ scored_findings | length }}</p>
    </div>

    {% for item in scored_findings %}
    {% if item.score.tier in ["publish", "revise"] %}
    <div class="finding">
        <h3>{{ item.title }}</h3>
        <span class="score-badge tier-{{ item.score.tier }}">
            {{ item.score.total }}/10 — {{ item.score.tier | upper }}
        </span>

        <p>{{ item.summary }}</p>

        <div class="dimensions">
            {% for dim_name, dim_score in item.score.dimensions.items() %}
            <div class="dim">{{ dim_name | title }}: {{ dim_score }}/10</div>
            {% endfor %}
        </div>

        {% if item.source %}
        <p><small>Source: {{ item.source }}</small></p>
        {% endif %}
    </div>
    {% endif %}
    {% endfor %}

    <footer>
        <p>Generated by KAZI OS v{{ metadata.kazi_version }} | Methodology v{{ metadata.methodology_version }}</p>
        <p>Scoring: {{ scoring.dimensions | map(attribute='name') | join(', ') }}</p>
    </footer>
</body>
</html>
```

## Markdown Template Example

```markdown templates/brief_v1.md
# {{ pipeline_name | title }} Brief — {{ date }}

**Client:** {{ org_id }}
**Run:** {{ run_id }}
**Items:** {{ scored_findings | length }} scored, {{ scored_findings | selectattr('score.tier', 'equalto', 'publish') | list | length }} publishable

---

{% for item in scored_findings %}
{% if item.score.tier == "publish" %}
## {{ item.title }}

**Score:** {{ item.score.total }}/10 ({{ item.score.tier }})

{{ item.summary }}

| Dimension | Score |
|-----------|-------|
{% for dim_name, dim_score in item.score.dimensions.items() %}
| {{ dim_name | title }} | {{ dim_score }}/10 |
{% endfor %}

{% if item.source %}
*Source: {{ item.source }}*
{% endif %}

---

{% endif %}
{% endfor %}

*Generated by KAZI OS | {{ date }}*
```

## JSON Export Template

```json templates/export_v1.json
{
  "report": {
    "pipeline": "{{ pipeline_name }}",
    "org_id": "{{ org_id }}",
    "run_id": "{{ run_id }}",
    "generated_at": "{{ date }}",
    "methodology_version": "{{ metadata.methodology_version }}"
  },
  "summary": {
    "total_scored": {{ scored_findings | length }},
    "tier_distribution": {
      {% for tier in scoring.tiers %}
      "{{ tier.name }}": {{ scored_findings | selectattr('score.tier', 'equalto', tier.name) | list | length }}{% if not loop.last %},{% endif %}
      {% endfor %}
    }
  },
  "findings": [
    {% for item in scored_findings %}
    {
      "title": "{{ item.title }}",
      "score": {{ item.score | tojson }},
      "summary": "{{ item.summary | replace('"', '\\"') }}"
    }{% if not loop.last %},{% endif %}
    {% endfor %}
  ]
}
```

## Rendering Reports

### In a Pipeline (automatic)

The synthesis agent renders templates automatically:

```python
class ReportCompiler(BaseAgent):
    name = "compiler"
    agent_type = "synthesis"

    async def execute(self, input: AgentInput) -> AgentOutput:
        template_path = self.config["template"]
        template = self.load_template(template_path)

        rendered = template.render(
            pipeline_name=input.context.pipeline_name,
            org_id=input.context.org_id,
            run_id=input.context.run_id,
            date=datetime.now().strftime("%Y-%m-%d"),
            scored_findings=input.payload["scored_findings"],
            metadata=input.payload.get("metadata", {}),
            scoring=self.scoring_config
        )

        return AgentOutput(
            status="success",
            payload={"report": rendered, "format": self.detect_format(template_path)}
        )
```

### Via CLI (manual)

```bash
# Render a report from a previous run
kazi render --run-id abc123 --template templates/report_v1.html --output report.html

# Render as PDF
kazi render --run-id abc123 --template templates/report_v1.html --format pdf --output report.pdf
```

## Custom Filters

Register custom Jinja2 filters for domain-specific formatting:

```python
# In your domain's __init__.py or routes.py
from kazi.delivery.templates import register_filter

@register_filter("currency")
def format_currency(value):
    return f"${value:,.2f}"

@register_filter("score_color")
def score_color(score):
    if score >= 7.5: return "#4ade80"
    if score >= 5.5: return "#fbbf24"
    if score >= 3.5: return "#fb923c"
    return "#ef4444"
```

Use in templates:

```html
<span style="color: {{ item.score.total | score_color }}">
    {{ item.score.total }}/10
</span>
<p>Estimated value: {{ item.value | currency }}</p>
```

## Multi-Format Output

A single pipeline can produce multiple output formats:

```yaml
# manifest.yaml
agents:
  - name: compiler
    type: synthesis
    config:
      templates:
        - path: templates/report_v1.html
          format: html
          destination: deliver
        - path: templates/brief_v1.md
          format: markdown
          destination: review
        - path: templates/export_v1.json
          format: json
          destination: archive
```

Each format routes to a different destination — the HTML goes to the client, the Markdown goes to Notion for review, the JSON goes to an archive.
